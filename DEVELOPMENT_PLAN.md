# GunBlade Development Plan

## Current Version Snapshot

Дата актуализации: 2026-04-14

Текущее состояние проекта уже вышло за рамки исходного плана "с нуля". Сейчас это не пустой прототип, а рабочий Godot 4 vertical slice со следующей базой:

- есть главное меню `scenes/menus/MainMenu.tscn`
- есть выбор персонажа `scenes/menus/CharacterSelect.tscn`
- есть глобальное состояние ранa в `scripts/core/Game.gd`
- есть 3 игровых архетипа через конфиги: `warrior`, `archer`, `monk`
- есть стартовая комната `scenes/game/levels/TestRoom.tscn`
- есть боевая комната `scenes/game/levels/Dungeon.tscn`
- игрок умеет двигаться, прыгать, атаковать в ближнем бою и стрелять
- враги умеют преследовать, атаковать, получать урон и умирать
- есть базовый HUD комнаты и возврат в меню по `Esc`
- есть простое сохранение выбранного персонажа и данных текущего ранa

Поэтому дальнейший план ниже построен не как "создать проект", а как план развития от текущей версии.

## Текущие Ограничения

Это то, что сейчас либо отсутствует, либо реализовано только как заглушка:

- нет настоящего перехода из стартовой комнаты в следующий боевой цикл
- нет генерации подземелья или цепочки комнат
- нет подбора лута и наград за зачистку
- нет полноценного progression loop между этажами
- нет босса и завершённого ранa
- нет полноценного game over flow
- нет инвентаря, экипировки, магазина, алхимии
- нет звука, музыки и hit feedback production-уровня
- нет автоматической тестовой/валидационной сборки

## Приоритетный План

### Phase 1: Stabilize Vertical Slice

Цель: довести текущий playable prototype до стабильного и понятного состояния, не расширяя резко объём игры.

- [ ] Проверить все `.tscn` на целостность и отсутствие битых ресурсов
- [ ] Синхронизировать `docs/README.md`, `DEVELOPMENT_PLAN.md` и фактическую структуру проекта
- [ ] Добить визуальную консистентность стартовой и боевой комнат
- [ ] Подчистить коллизии платформ, спавны и безопасные позиции игрока/врагов
- [ ] Отбалансировать параметры `warrior`, `archer`, `monk`
- [ ] Довести поведение стрелы и ближних атак до предсказуемого feel
- [ ] Убрать явные debug/former-test артефакты из UI и сцен

### Phase 2: Build Actual Run Flow

Цель: превратить набор отдельных сцен в минимальный, но законченный loop.

- [ ] Добавить явный переход из стартовой комнаты в боевую
- [ ] Добавить условие зачистки боевой комнаты
- [ ] Добавить переход на следующий floor или следующую комнату
- [ ] Связать `Game.current_floor` с реальным прогрессом между сценами
- [ ] Решить, остаётся ли `TestRoom.tscn` стартовым хабом или будет переименован в `StartRoom.tscn`

### Phase 3: Expand Combat Depth

Цель: сделать бой не только рабочим, но и вариативным.

- [ ] Добавить 2-3 типа врагов поверх текущего pawn enemy
- [ ] Ввести различия по поведению: rush, guard, ranged, elite
- [ ] Добавить телеграфы атак врагов
- [ ] Добавить hit flash / knockback / death feedback
- [ ] Пересмотреть систему контакта врага с игроком, чтобы бой не ощущался сырым

### Phase 4: Introduce Core Roguelike Rewards

Цель: начать формировать мотивирующую награду за бой.

- [ ] Добавить награду за зачистку комнаты
- [ ] Добавить базовый gold loop
- [ ] Подготовить каркас под `runes`, `curses`, `relics`, которые уже есть в `current_run_data`
- [ ] Показать награду игроку через простой post-room UI

### Phase 5: Meta Systems

Цель: расширить игру за пределы одной боевой сцены.

- [ ] Инвентарь
- [ ] Экипировка
- [ ] Магазин между боями
- [ ] Алхимия/расходники
- [ ] Экран смерти и статистика ранa
