# Progress

## Status

- Memory Bank инициализирован с нуля на основе фактического состояния репозитория.
- Создан `docs/README.md` как канонический источник высокоуровневой архитектуры.
- Добавлен канонический список deliverables в `memory_bank/projectbrief.md` с суммой весов 100.
- Исправлена причина смещения игрока при движении: `Player.gd` больше не режет все анимации как 8-кадровые.
- Добавлено отдельное состояние анимации атаки игрока, чтобы attack texture и кадры проигрывались полностью.
- Добавлен визуальный slash-эффект удара игрока через `Line2D`.
- В игровую сцену добавлены простое окружение арены, полоска здоровья игрока, статичный враг с полоской здоровья, контактным уроном и респавном.

## Known Issues

- В репозитории отсутствовал `docs/README.md`, хотя правила AGENTS.md требуют считать его источником архитектурной правды.
- `scripts/systems/GameManager.gd` выглядит несинхронизированным с текущим API `Player.gd` и может быть legacy-артефактом.
- Реальный код покрывает только ранний прототип и существенно отстает от полного PRD/MVP scope.
- `scenes/game/characters/Player.tscn` все еще выглядит устаревшей относительно текущей реализации игрока в `scenes/game/levels/Dungeon.tscn`.
- `scenes/shared/HealthBar.tscn` пока не используется в runtime: полоски здоровья заданы локально в сценах игрока и врага.
